# 贡献指南

感谢改进 `delivery-loop`。本项目优先保持 skill 简洁、可验证、可安装。

## 修改原则

- 修改 `SKILL.md` 时，保持流程可执行，避免加入泛泛而谈的提示词。
- 新增资源目录前先确认是否会被 agent 实际使用；无必要不新增 `assets/`、`references/` 或 skill 运行时 `scripts/`。
- 修改安装逻辑时，禁止删除用户已有目录或读取敏感信息。
- 所有文档和注释使用简体中文。

## 提交前检查

```bash
npm test
```

检查通过后再提交 PR，并在说明中写明动机、变更范围、验证结果和风险。
